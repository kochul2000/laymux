# 0040. Remote 소프트 키는 사용자가 정한 순서를 유지한다

- Status: Accepted
- Date: 2026-07-20
- Source: 사용자 요구(Keys로 추가한 키의 순서 변경); [ADR-0028](0028-remote-soft-key-toolbar.md); [architecture/api-contracts.md §12](../architecture/api-contracts.md)
- Extends: [ADR-0028](0028-remote-soft-key-toolbar.md)

## Context

ADR-0028은 Remote 소프트 키 툴바의 활성 세트와 커스텀 키를 합친 뒤 전역 고정 순서로 렌더하도록 정했다. 따라서 사용자가 개별 키를 추가하는 순서는 저장 배열에 남더라도 실제 툴바 순서에는 반영되지 않으며, 자주 쓰는 키를 엄지손가락이 닿기 쉬운 위치로 옮길 수 없다.

순서는 Remote 브라우저 한 surface의 사용성에 관한 상태다. PTY 입력 바이트, Remote API, 데스크톱 설정과는 무관하며 기존 `laymux.remote.keybar` localStorage 소유권을 유지해야 한다. 기존 저장 값에는 순서 필드가 없으므로 업데이트 직후 현재 기본 순서가 바뀌어서는 안 된다.

이번 결정의 범위는 이미 제공되는 소프트 키의 표시 순서다. 키 정의, escape sequence, 키 세트 구성, 기기 간 동기화는 바꾸지 않는다.

## Decision

Remote 소프트 키 툴바는 **활성 키 전체를 대상으로 사용자가 정한 순서를 surface-local 상태로 저장하고 그대로 렌더한다.**

- `laymux.remote.keybar`에 알려진 전체 키 ID의 `order`를 저장한다. 표시 목록은 활성 세트와 커스텀 키의 합집합을 구한 뒤 `order`로 필터링해 계산한다.
- 설정 팝오버의 마지막 섹션에 현재 표시 중인 키를 실제 툴바 순서와 같은 왼쪽→오른쪽 흐름의 컴팩트 칩 그리드로 보여준다. 정렬 칩은 커스텀 키 팔레트의 원본 키 칩과 동일한 크기·패딩·타이포그래피를 재사용한다.
- 칩을 길게 누른 뒤 드래그하는 조작을 기본 재정렬 경로로 사용하고, 드래그 중 삽입 위치를 명시적인 표시선으로 보여준다. HTML native drag-and-drop 대신 Pointer Events를 사용해 모바일 touch와 desktop mouse를 같은 경로로 처리한다.
- 칩 탭은 키를 선택하며 `맨 앞`·`왼쪽`·`오른쪽`·`맨 뒤` 보조 동작을 노출한다. 드래그를 쓰기 어려운 키보드·보조 기술 사용자도 같은 순서를 만들 수 있어야 한다.
- `Reset`은 전체 순서를 정본 `KEY_ORDER`로 되돌린다. 모든 이동과 초기화 결과는 즉시 툴바와 localStorage에 반영한다.
- 현재 비활성인 커스텀 키를 새로 선택하면 기존 활성 키의 맨 뒤에 붙인다. 이미 세트로 활성인 키를 커스텀으로도 선택하는 경우에는 표시 순서를 바꾸지 않는다.
- 비활성 키도 전체 `order`에는 유지한다. 세트나 커스텀 키를 잠시 해제했다가 다시 켜도 사용자가 정한 상대 위치를 잃지 않는다.
- 저장된 순서의 알 수 없는 ID와 중복 ID는 읽을 때 제거하고, 새로 추가된 키 정의는 정본 `KEY_ORDER` 순서로 빠진 항목 뒤에 보충한다.
- `order`가 없는 기존 저장 값은 현재 `KEY_ORDER`를 초기 순서로 사용한다. 별도 마이그레이션 로직이나 버전 필드는 두지 않는다.
- 이 상태는 계속 Remote 클라이언트 전용이다. Remote API, PTY 입력 경로, `settings.json` 스키마는 변경하지 않는다.

## Alternatives Considered

- **고정 `KEY_ORDER` 유지**: 구현은 단순하지만 사용자가 추가한 키의 위치를 조정하려는 요구를 해결하지 못한다.
- **커스텀 키끼리만 순서 변경**: 세트 키보다 커스텀 키를 앞으로 옮길 수 없어 실제 툴바 배치 제어가 불완전하다.
- **활성 키마다 세로 행과 좌우 버튼 제공**: 구현과 접근성은 단순하지만 목록이 길어지고, 여러 칸 옮길 때 버튼을 반복해서 눌러야 한다. 초기 프로토타입에서 기능은 동작했지만 실제 모바일 사용성이 낮아 채택하지 않았다.
- **실제 하단 툴바에서 직접 드래그**: 결과를 그대로 보며 조작할 수 있지만 키 전송 탭 및 가로 스크롤과 충돌해 오입력 위험이 크다. 입력을 보내지 않는 설정 팝오버 안의 미리보기만 재정렬 surface로 사용한다.
- **드래그만 제공**: 빠르지만 정밀 pointer 조작이 어려운 사용자와 키보드·보조 기술 사용자를 배제한다. 선택 후 보조 이동 동작을 함께 제공한다.
- **`settings.json`에 동기화**: 기기 간 공유는 가능하지만 surface-local 표시 상태라는 ADR-0004/0028의 경계를 넓히며, 현재 요구에 비해 계약과 동기화 복잡도가 크다.

## Consequences

- 사용자는 세트 키와 개별 추가 키를 섞어 한 번의 드래그로 먼 위치까지 옮길 수 있고, 새로고침과 세트 토글 뒤에도 배치가 유지된다.
- 컴팩트 그리드는 세로 행 목록보다 적은 공간에 전체 순서를 보여주지만, 실제 한 줄 툴바가 그리드에서는 여러 줄로 접힌다. 읽기 순서는 항상 왼쪽→오른쪽, 위→아래로 정의한다.
- 로컬 저장 값은 알려진 전체 키 ID를 담으므로 이전보다 조금 커지지만 크기는 키 정의 수에 한정된다.
- 새 키 정의를 추가할 때 `KEY_ORDER`에도 등록해야 기존 클라이언트 저장 값에 결정론적으로 합쳐진다.
- Playwright 테스트는 키 추가, long-press drag, 삽입 표시, 선택형 보조 이동, Reset, 즉시 렌더, localStorage 저장, 새로고침 복원을 검증한다. 정적 Remote page 테스트는 순서 계산과 조작 UI의 존재를 검증한다.
- 향후 기기 간 동기화가 필요하면 상태 소유권을 별도로 재검토한다.
